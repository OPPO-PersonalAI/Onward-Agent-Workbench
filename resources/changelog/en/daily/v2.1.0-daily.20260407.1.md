# Onward Daily Build v2.1.0-daily.20260407.1

Changes since `v2.1.0-daily.20260406.1`.

## New Features
- Add in-app feedback workflow backed by GitHub Issues, with local history persistence and public issue status tracking.
- Add a packaged Change Log viewer accessible from the sidebar, displaying release notes for the current build.
- Add real-time image change detection in Markdown preview with automatic cache invalidation and a manual refresh button.
- Introduce a shared dropdown component with consistent sizing and cross-platform arrow alignment across all select controls.

## Bug Fixes
- Fix terminal PATH resolution on macOS and Linux by launching the default shell as a login shell, ensuring profile files are sourced correctly.
- Prevent data loss during app quit and auto-update by flushing all pending debounced saves before exit.
- Add UI preference persistence across upgrades so panel widths, toggle states, and viewer settings survive version transitions.
- Unify project editor file memory into per-file state, fixing stale reopen tracking, markdown preview anchor restoration, and legacy data migration.
- Persist project editor sidebar scroll positions for both file browser and outline panels, surviving tab switches and app restarts.
- Improve scroll restore reliability with extended retry attempts and stale quick-file list validation to prevent async operations from overwriting user changes.
- Show an explicit "Already up to date" confirmation in Settings after a manual update check completes with no available update.

## UI Improvements
- Redesign sidebar icons to a unified 20×20 / stroke-width-2 style with new sparkles (Change Log) and lightbulb (Feedback) icons.
- Replace Change Log modal backdrop blur with a solid dark overlay for improved rendering performance.
